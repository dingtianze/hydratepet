# PRD框架：智能喝水提醒网页应用
## 产品名暂定：「水时刻」/「HydrateMe」

---

## 一、调研数据分析

### 1.1 用户画像（基于问卷）

| 维度 | 数据 |
|------|------|
| **年龄段** | 18-25岁（年轻群体） |
| **身份** | 学生 + 办公室白领（跨场景用户） |
| **主要设备** | 只用手机 ⚠️ |
| **当前饮水量** | 1-2杯/天（200-400ml）——严重不足！ |

### 1.2 核心痛点验证

| 痛点 | 用户选择 | 验证结果 |
|------|----------|----------|
| 经常忘记喝水 | ✅ | 确认存在 |
| 口渴才想起 | ✅ | 被动触发，缺乏主动提醒 |
| 社交触发 | ✅（同事去接水时跟着去） | 社交因素重要 |

### 1.3 ⚠️ 关键发现

#### 发现1：网页端接受度存疑
用户期望的形式：**手机App + 桌面小组件 + 智能手环**
- 没有选择「网页版」
- 用户主要设备是手机

#### 发现2：游戏化需求强烈 💡
用户主动提出的创意：
> "通过喝水，来搞一个像素宠物养成，根据每天、每月喝水规律生成宠物形象，并给用户生成网络搞笑称号，例如沙漠骆驼，深海霸主等"

#### 发现3：功能偏好
- 看重：个性化饮水量推荐 + 游戏化元素
- 提醒方式：工作时间内均匀间隔
- 登录：可接受手机号/微信登录

#### 发现4：商业化
- 倾向免费，但愿意为好功能付费

## 二、需求调整建议

### 2.1 产品形态调整

**原方案**：纯网页端
**建议调整为**：

```
┌─────────────────────────────────────────────┐
│              产品形态：PWA 网页应用            │
│  （Progressive Web App - 渐进式网页应用）     │
├─────────────────────────────────────────────┤
│  优势：                                      │
│  ✓ 无需下载安装，浏览器打开即用               │
│  ✓ 可添加到手机桌面，像App一样使用            │
│  ✓ 支持推送通知（提醒功能）                   │
│  ✓ 跨平台（PC + 手机）                        │
│  ✓ 可离线使用（基础功能）                     │
└─────────────────────────────────────────────┘
```

### 2.2 核心功能优先级（MVP）

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 像素宠物养成系统 | 用户主动提出的核心创意，差异化亮点 |
| P0 | 饮水记录与目标 | 基础功能，用户已选 |
| P0 | 定时提醒 | 解决「忘记喝水」痛点 |
| P1 | 个性化饮水量推荐 | 用户看重功能 |
| P1 | 搞笑称号系统 | 增强趣味性，社交分享点 |
| P2 | 数据统计图表 | 长期留存 |
| P2 | 社交/打卡功能 | 增强粘性 |

## 三、核心功能设计

### 3.1 像素宠物养成系统（核心差异化）

#### 宠物形象生成逻辑

```
初始状态：
┌──────────────┐
│   🥚 蛋      │  ← 新用户注册后获得宠物蛋
└──────────────┘

根据喝水数据进化：

每日喝水达标 → 宠物成长值+1
连续达标7天  → 进化形态1
连续达标30天 → 进化形态2

形象变化维度：
├── 体型（喝水越多越饱满）
├── 颜色（根据喝水时间分布变化）
├── 装饰（解锁成就获得配饰）
└── 表情（健康/口渴/开心状态）
```

#### 称号系统

| 称号 | 获得条件 | 趣味性 |
|------|----------|--------|
| 🏜️ 沙漠骆驼 | 连续3天喝水<500ml | 自嘲式激励 |
| 🐠 浅水鱼 | 喝水达标但时间不规律 | 温和提醒 |
| 🌊 深海霸主 | 连续30天达标 | 成就感 |
| ⏰ 喝水机器 | 每天准时准点喝水 | 规律性表扬 |
| 🧊 冰山美人 | 只喝冰水 | 习惯标签 |
| 🌵 仙人掌 | 周末也坚持喝水 | 自律认证 |

### 3.2 饮水提醒机制

```
默认提醒策略：
├── 工作日：9:00-18:00
│   ├── 上午：9:00 / 10:30 / 12:00
│   ├── 下午：14:00 / 15:30 / 17:00
│   └── 下班前：17:30
├── 周末：根据用户习惯智能调整
└── 提醒方式：浏览器通知 + 宠物动画提醒

个性化设置：
├── 可自定义工作时间
├── 可调整提醒频率
├── 可关闭休息日提醒
└── 可设置免打扰时段
```

### 3.3 个性化饮水量推荐

```
计算公式（基础版）：
每日建议饮水量 = 体重(kg) × 30-35ml

调整因子：
├── +300ml：运动日
├── +200ml：天气炎热
├── -200ml：久坐少动日
└── 女性：基础值 × 0.9

简化交互：
用户输入：体重 + 性别
系统输出：每日目标 + 每次建议量
```

## 四、用户旅程设计

### 4.1 首次使用流程

```
Step 1: 打开网页
    ↓
Step 2: 孵化宠物蛋 🥚
    │   "你好！我是你的喝水小助手，
    │    快来孵化我吧～"
    ↓
Step 3: 设置个人信息
    │   - 体重/性别（计算饮水量）
    │   - 工作时间（设定提醒时段）
    │   - 登录方式（手机号/微信）
    ↓
Step 4: 完成首次喝水记录
    │   "恭喜你！你获得了『初出茅庐』称号！"
    ↓
Step 5: 添加到桌面（PWA引导）
    "添加到桌面，像App一样方便使用～"
```

### 4.2 日常使用流程

```
每日循环：

上班打开 → 看到宠物状态
    ↓
收到提醒 → 浏览器弹窗 + 宠物动画
    ↓
点击记录 → 一键打卡/选择饮水量
    ↓
宠物反馈 → 成长动画 + 鼓励语音
    ↓
查看进度 → 今日目标完成度
    ↓
获得称号 → 分享到朋友圈
```

## 五、技术方案建议

### 5.1 技术栈

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| 前端 | React/Vue + PWA | 组件化开发，支持离线 |
| 后端 | Node.js/Python | 快速开发 |
| 数据库 | MongoDB/PostgreSQL | 用户数据存储 |
| 通知 | Service Worker + Push API | 浏览器推送 |
| 部署 | Vercel/Netlify | 免费托管 |

### 5.2 关键功能实现

```
PWA核心功能：
├── manifest.json    → 配置桌面图标、主题色
├── service-worker   → 离线缓存、后台推送
└── push API         → 定时提醒

像素宠物实现：
├── 状态管理：Redux/Vuex 存储宠物数据
├── 形象生成：CSS/SVG 动态组合
└── 动画效果：Lottie/GSAP
```

## 七、风险评估与应对

| 风险 | 影响 | 应对方案 |

| 竞品模仿 | 中 | 快速迭代，建立品牌认知 |
